1 / Les techniques de l'information et de la communication (TIC) :

Les techniques de l'information et de la communication (TIC) regroupent les outils utilisés pour manipuler, traiter, produire, transmettre et échanger des informations. . Les TIC permettent la communication, l'accès aux sources d'informations le stokage la manipulation et la production d'informations sous forme de texte, image, son et vidéo. Elles font appel à des techniques telles que l'informatique, Internet, l'audiovisuel, le multimédia et les télécommunications.

2/Les TIC sont regroupées en trois grandes catégories :

la téléinformatique, les télécommunications et l'électronique grand public.

La téléinformatique :

concerne l'échange d'informations numériques entre systèmes informatiques, incluant Internet, les serveurs, les réseaux, etc.

Les télécommunications

englobent les systèmes communicants qui ne sont pas nécessairement informatiques, tels que le traitement du signal, la transmission analogique, la téléphonie mobile, etc.

L'électronique

grand public concerne les composants et équipements électroniques utilisés par le grand public, tels que les télévisions, les lecteurs DVD, etc.

Les objectifs recherchés dans le domaine des TIC sont les suivants :

La connectivité : permettre la communication entre différents types d'ordinateurs utilisant des logiciels différents. La modularité : utiliser un ensemble restreint d'appareils généraux qui peuvent être adaptés à différentes configurations. Une implantation simple : proposer des solutions général.

3/Les composants des TIC se divisent en deux parties
La partie matérielle

comprend les dispositifs physiques utilisés dans les TIC, tels que : Les ordinateurs portables, les télévisions, les téléphones portables, les tablettes interactives, les appareils photo numériques, les robots, etc. Les stations de travail, qui sont des machines capables d'envoyer des données vers les réseaux, tels que les ordinateurs personnels (PC), les ordinateurs Apple (MAC), les terminaux SUN (Terminal X), les serveurs, etc.

La partie logicielle

englobe les logiciels utilisés dans les TIC, notamment : Les systèmes d'exploitation, tels que Windows, macOS, Linux, etc., qui gèrent les ressources matérielles et permettent l'exécution des applications, les logiciels d'application, de retouche d'image et de montage vidéo, etc.

4/ Les TIC présentent plusieurs caractéristiques qui les rendent bénéfiques pour les entreprises :

* Rapidité

* Décentralisation du pouvoir,

* Meilleur accès aux informations,

* Rationalité .

Objectifs de communication : Les objectifs de communication sont les réactions attendues du destinataire. 1. L’histoire de l’internat :

L’internet est très important dans notre vie quotidienne il a été inventé le 1 Janvier 1983, puis s'est répandu dans le monde entier pour devenir accessible à tous vers 1990. L'internet est un ensemble de réseaux mondiaux interconnectés qui permet á des ordinateurs et à des serveurs de communiquer efficacement au moyen d'un protocole de communication comme elle est important dans notre vie elle a bcp des avantages

2. Un réseau informatique :

est composé d'ordinateurs et de terminaux connectés entre eux, permettant l'échange d'informations numériques selon des règles définies. La mise en réseau consiste à relier des ordinateurs pour qu'ils partagent des ressources. Les réseaux reposent sur l'interconnexion, assurant la transmission des données d'un nœud à un autre, et la communication, permettant l'échange de données entre processus.

3. Les caractéristiques d'un réseau

incluent sa topologie physique et logique, le débit (bits/s), la distance maximale et le nombre maximum de nœuds interconnectés.

4. Quelques termes importants en réseau :

sont la station de travail, le nœud, le serveur et le paquet. Les réseaux peuvent être homogènes (même constructeur) ou hétérogènes (divers constructeurs).

5. Les éléments d'un réseau :

comprennent les ordinateurs équipés de cartes de communication, les logiciels et les supports de connexion.

6. Classification des réseaux :

Les réseaux peuvent être classés selon leur portée (LAN, MAN, WAN) et leur utilisation (intranet, extranet, Internet). Les supports de connexion peuvent être filaires (coaxial, paires torsadées, fibre optique) ou sans fil (ondes radios, infrarouges, satellites, Bluetooth). Les réseaux peuvent également être de type peer-to-peer ou client-serveur.

7. La topologie d'un réseau :

peut-être en bus, en étoile, en anneau ou maillée. Chacune a ses propres caractéristiques et méthodes de transmission des données.

En résumé :

un réseau informatique est un ensemble d'ordinateurs interconnectés pour permettre la communication et l'échange de données, avec diverses configurations et technologies adaptées à différents besoins

Chapitre 3 :Le modèle OSI


  • Mammeri Nour Djihane
  • Refsi Meriem

Les composants matériels d'un réseau informatique comprennent les cartes réseau et les switches.

Les cartes réseau, également connues sous le nom de cartes d'interface réseau (NIC), sont des composants installés à l'intérieur d'un ordinateur qui lui permettent de se connecter à un réseau, qu'il s'agisse d'Internet ou d'un réseau local. Elles sont généralement connectées à une prise Ethernet ou Wi-Fi. Les cartes réseau jouent un rôle crucial dans la communication entre les ordinateurs d'un réseau. Elles permettent à un ordinateur de transmettre et de recevoir des données en utilisant des protocoles de communication spécifiques tels que TCP/IP, qui sont utilisés pour transférer des données sur Internet.

Les switches

également appelés commutateurs réseau, sont des composants matériels utilisés pour connecter des appareils sur un réseau local (LAN). Les switches permettent à plusieurs appareils, tels que des ordinateurs, des imprimantes et des serveurs, de communiquer entre eux en utilisant des adresses MAC (Media Access Control). Lorsqu'un appareil se connecte à un switch, celui-ci vérifie son adresse MAC et stocke cette information dans une table de correspondance. Lorsqu'un autre appareil du réseau souhaite communiquer avec cet appareil, le switch utilise cette table pour envoyer les données uniquement à l'appareil destinataire, plutôt que de les diffuser à tous les appareils du réseau.

Les switches sont disponibles en différentes tailles, allant des petits switches pour les réseaux domestiques et les petits bureaux, aux switches de niveau entreprise pour les réseaux plus importants. Ils peuvent offrir des fonctionnalités avancées telles que la gestion des VLAN, la qualité de service (QoS), la gestion du trafic, etc.

Ces deux composants sont essentiels pour la mise en place d'un réseau informatique fonctionnel, permettant la communication et le partage de données entre les différents appareils connectés.

Un hub

est un composant matériel utilisé pour connecter plusieurs appareils sur un réseau local (LAN). Il fonctionne comme un répéteur en recevant les données envoyées par un appareil et les transmettant à tous les autres appareils connectés. Contrairement à un switch, le hub ne possède pas la fonctionnalité de table de correspondance, ce qui signifie que toutes les données envoyées sur le réseau sont reçues par tous les appareils connectés, même si l'appareil destinataire est différent. Les hubs ont été largement remplacés par les switches, qui offrent une gestion du trafic plus efficace, une communication plus rapide et plus sécurisée entre les appareils connectés. Cependant, les hubs peuvent encore être utilisés dans des cas spécifiques ou dans de petits réseaux où la complexité d'un switch n'est pas nécessaire. Le modèle OSI (Open Systems Interconnection) est un modèle de référence pour les communications réseau. Il divise les communications en sept couches distinctes, chacune étant responsable de fonctions spécifiques liées à la communication de données entre les appareils du réseau. Le modèle OSI est souvent utilisé pour la conception et la mise en œuvre de réseaux informatiques.

Les sept couches du modèle OSI sont :

1. La couche physique : responsable de la transmission des données sur le support physique.

2. La couche liaison de données : responsable de la transmission des données entre les appareils partageant le même support physique.

3. La couche réseau : responsable du routage des données à travers le réseau.

4. La couche transport : responsable de l'acheminement des données de bout en bout entre les appareils.

5.Le Modèle TCP/IP

Le modèle TCP/IP (Transmission Control Protocol/Internet Protocol) est un modèle de référence largement utilisé dans les communications réseau. Contrairement au modèle OSI, qui comprend sept couches, le modèle TCP/IP se compose de quatre couches principales :

1. La couche liaison de données : Cette couche est responsable de la transmission des données sur le support physique, tel que le câble, la fibre optique ou les ondes radio. Elle gère les protocoles qui définissent la structure des trames et assure la fiabilité de la transmission.

2. La couche réseau : Cette couche est chargée du routage des données à travers le réseau. Elle utilise des adresses IP pour déterminer la meilleure route à emprunter afin de livrer les paquets de données à leur destination.

3. La couche transport : Cette couche est responsable de l'acheminement des données de bout en bout entre les appareils. Elle garantit la fiabilité et le contrôle de flux des données en utilisant les protocoles TCP (Transmission Control Protocol) et UDP (User Datagram Protocol).

4. La couche application : Cette couche permet aux applications de communiquer entre elles via le réseau. Elle inclut des protocoles tels que HTTP (Hypertext Transfer Protocol), FTP (File Transfer Protocol), SMTP (Simple Mail Transfer Protocol) et DNS (Domain Name System). Le modèle TCP/IP est largement utilisé dans l'architecture d'Internet et sert de base pour les protocoles et les communications sur le Web. Il facilite l'interconnexion des réseaux et la transmission efficace des données à travers différentes technologies et plates-formes. Il y’a deux mode d’adressage IPV4 et IPV6 : 1. ADRESSE IPV4: a. Chaque paquet contient : adresse source sur (32 bits) et une adresse destinataire sur (32bits)

L'adresse IPv4 est un numéro unique attribué à chaque appareil sur un réseau qui utilise le protocole Internet. Une adresse IPv4 est composée de 32 bits, ce qui équivaut à 4 octets. Chaque octet est généralement représenté en décimal et séparé par un point, formant ainsi une adresse IPv4 sous la forme xxx.xxx.xxx.xxx. Par exemple, l'adresse IPv4 172.16.4.20 est représentée en binaire comme 10101100 00010000 00000100 00010100, et en décimale comme 172.16.4.20.

Dans une adresse IPv4, les 32 bits sont divisés en deux parties : la partie réseau et la partie hôte. La partie réseau identifie le réseau auquel l'appareil est connecté, tandis que la partie hôte identifie l'appareil spécifique sur ce réseau.

Il existe différents types d'adresses IPv4, tels que l'adresse réseau et l'adresse de diffusion. L'adresse réseau est caractérisée par tous les bits de la partie hôte étant égaux à zéro. Par exemple, pour l'adresse réseau 10.0.0.0, tous les hôtes de ce réseau auront la même partie réseau. L'adresse de diffusion, quant à elle, est caractérisée par tous les bits de la partie hôte étant égaux à un. Par exemple, pour le réseau 10.0.0.0, l'adresse de diffusion serait 10.255.255.255. L'adresse de diffusion permet à un appareil d'envoyer des paquets à tous les hôtes du réseau simultanément, sans avoir à les adresser individuellement.

Il convient de noter que l'espace d'adressage IPv4 est limité en raison du nombre fini de combinaisons possibles avec 32 bits. En raison de cette limitation, le protocole IPv6 a été développé pour fournir un espace d'adressage plus vaste avec ses 128 bits, permettant ainsi de répondre aux besoins croissants de connectivité à l'ère de l'Internet des objets et de l'expansion d'Internet.

Dans un réseau IPv4, l'adresse hôte fait référence à la partie de l'adresse IP qui identifie une machine spécifique sur le réseau. L'attribution des valeurs de l'adresse hôte se situe entre l'adresse réseau et l'adresse de diffusion.

Prenons l'exemple d'un réseau avec l'adresse réseau 10.0.0.0 et l'adresse de diffusion 10.255.255.255. Les adresses des machines (hôtes) sur ce réseau vont de : • 10.0.0.1 : première adresse hôte (machine) • ... • 10.255.255.254 : dernière adresse hôte (machine) La plage d'adresses entre l'adresse réseau et l'adresse de diffusion est réservée aux machines individuelles connectées à ce réseau. Chaque machine se voit attribuer une adresse hôte unique pour identifier de manière unique son emplacement sur le réseau. Le masque de réseau est utilisé pour déterminer quelle partie de l'adresse IP correspond à la partie réseau et quelle partie correspond à la partie hôte. Dans l'exemple donné, le masque de réseau est 255.0.0.0, ce qui signifie que les 8 premiers bits de l'adresse IP sont réservés à la partie réseau et les 24 derniers bits sont réservés à la partie hôte. L'adresse réseau (10.0.0.0) identifie le réseau dans son ensemble, tandis que l'adresse de diffusion (10.255.255.255) est utilisée pour envoyer des paquets à tous les hôtes du réseau simultanément. En utilisant le masque de réseau, il est possible de définir des sous-réseaux, ce qui permet de diviser un réseau en sous-groupes logiques plus petits. Chaque sous-réseau aurait alors son propre réseau, adresse de diffusion, plage d'adresses d'hôtes et masque de sous-réseau correspondants.

1-introduction :

Le Web et Internet c’est pareil : Le Web et Internet sont étroitement liés. Le World Wide Web (WWW) est un système hypertexte public qui fonctionne sur Internet. Il permet de consulter des pages accessibles sur des sites en utilisant un navigateur. Les pages web sont reliées les unes aux autres par des hyperliens, d'où l'analogie de la toile d'araignée. Le Web est une application d'Internet, qui est un support de communication général.

Le principe du World Wide Web : Le principe du World Wide Web consiste à accéder à des documents textuels situés sur des machines accessibles via Internet et liés entre eux par des liens hypertexte. Actuellement, le Web sert à fournir des ressources de différentes natures, telles que du texte, des images, du son, de la vidéo, du contenu applicatif interactif comme les achats en ligne ou les discussions.

Une URL (Uniform Resource Locator) est un lien hypertexte qui permet de suivre une adresse spécifique et d'ouvrir une nouvelle page. Elle décrit l'emplacement d'une ressource en fournissant l'adresse du serveur et l'adresse de la page sur le serveur.

2-Les aspects techniques du Web :

1. Les URL : permettent d'accéder à un ensemble de ressources en utilisant le protocole DNS pour la localisation.

2. http est un protocole de niveau applicatif utilisé dans le paradigme client-serveur.

3. HTML est un langage de description de pages web qui permet d'intégrer du texte, des images et d'autres objets..

3-page web et site web :

Une page web est un document multimédia composé d'images, de textes et offrant la possibilité d'interactions et de liens. Elle est décrite élément par élément, tels que le titre, les morceaux de texte et les images, à l'aide d'un langage de description comme HTML, XHTML ou HTML5.

4- La création d'une page web HTML :

La création d'une page web HTML implique l'utilisation du langage de description HTML, qui est un fichier texte structuré par des balises. Un fichier HTML doit contenir au minimum les balises <html>, <head>, <title> et <body>. . Le texte présente des exemples de balises HTML pour structurer un document, ainsi que les éléments de base qui doivent être inclus dans tout fichier HTML, tels que la balise <html>, <head>, <title> et <body>. 1.Qu’est-ce que Git ?

Les logiciels de gestion de versions sont quasiment incontournables aujourd`hui car ils facilitent grandement la gestion de projets et car ils permettent de travailler en équipe de manière beaucoup plus efficace.

Parmi les logiciels de gestion de versions, Git est le leader incontesté et il est donc indispensable pour tout développeur de savoir utiliser Git

2. A quoi sert concrètement un système de gestion de version ?

L`organisation du travail est ici beaucoup plus complexe puisqu`il va falloir s`assurer : • Que les fonctionnalités sur lesquelles travaillent un développeur ne rentrent pas en conflit avec les fonctionnalités sur lesquelles travaillent des autres développeurs ; • Que chaque développeur sache sur quoi travaillent les autres développeurs afin de travailler correctement ;• que chaque développeur possède une version actualisée du site pour tester et implémenter ses fonctionnalités.

Ici, l`idée la plus logique serait de mettre en place un serveur distant qui contiendrait l`historique des modifications faites par chaque développeur afin que chacun ait accès aux avancées des autres.

Grosso-modo, les logiciels de versioning et Git en particulier vont nous permettre d`effectuer ces opérations mais de manière beaucoup plus robuste et avec de nombreuses fonctionnalités supplémentaires très utiles (possibilité de synchroniser son dossier local avec le serveur, connexion aux autres projets et mise à jour des différences automatisée, etc.

Git permet de coordonner le travail entre plusieurs personnes en conservant un historique des changements effectués sur des fichiers. Les développeurs travaillant avec Git.

Modèle centralisé vs modèle décentralisé

Le modèle distribué a été popularisé par Git et présente différents avantages notables par rapport au modèle centralisé : • Simplicité / flexibilité du travail : Comme chaque utilisateur peut héberger le code du projet, on n`a plus besoin d`être constamment connecté à un serveur central et on peut donc travailler en ligne sur sa propre machine ;• Sécurité : Comme chaque utilisateur possède le code complet d`un projet, on peut utiliser la copie du projet d`un utilisateur comme back- up en cas de corruption du serveur central

Git est un logiciel de gestion de version. GitHub est un service en ligne qui permet d`héberger des dépôts ou repo Git.

10. Les états des fichiers :

Un fichier possède l`état “suivi” s`il appartenait au dernier instantané capturé par Git, c`est-à-dire si il est enregistré en base. Lorsqu`on démarre un dépôt Git en important un répertoire déjà existant depuis notre machine, les fichiers sont au départ tous non suivis. Lorsqu`on clone un dépôt Git déjà existant, c`est différent puisqu`on copie tout l`historique du projet et donc les fichiers sont tous déjà suivis par défaut. Ensuite, chaque fichier suivi peut avoir l`un de ces trois états : • Modifié (“modified”) ; • Indexé (“staged”) ; • Validé (“committed”). Lors du clonage d`un dépôt déjà existant, les fichiers sont enregistrés par défaut en base et ils sont donc validés par défaut. Les fichiers modifiés vont être considérés comme “modifiés” par Git tandis que les nouveaux fichiers vont être “non suivis”. Un fichier modifié est considéré comme “modifié” par Git tant qu`il n`a pas été indexé

11.Les zones de travail :

Les états de fichiers sont liés à des zones de travail dans Git. En fonction de son état, un fichier va pouvoir apparaitre dans telle ou telle zone de travail. Tout projet Git est composé de trois sections : le répertoire de travail (working tree), la zone d`index (staging area) et le répertoire Git (repository). La zone d`index ou “staging area” correspond à un simple fichier, généralement situé dans le répertoire Git, qui stocke les informations concernant ce qui fera partie du prochain instantané ou du prochain “commit”. Le répertoire Git est l`endroit où Git stocke les méta-données et la base de données des objets de votre projet. E que son nom est ajouté à la zone d`index, il possède l`état indexé. Finalement, on va valider (“commit”) la version indexée de nos fichiers pour les ajouter au répertoire Git.

14.Ajouter ou modifier des fichiers dans un projet et Actualiser notre dépôt Git :

A ce niveau, nous avons donc d`un côté notre projet contenant un ensemble de fichiers et de ressources sur lesquelles on travaille ainsi qu`un dépôt Git qui sert à gérer les différentes versions de notre projet. On va indiquer tous ces changements à Git pour qu`il conserve un historique des versions auquel on pourra ensuite accéder pour revenir à un état précédent du projet (dans le cas où une modification entraine un bogue par exemple ou n`amène pas le résultat souhaité). A chaque fois qu`on souhaite enregistrer une modification de fichier ou un ajout de fichier dans le dépôt Git, on va devoir utiliser les commandes git add et git commit comme on a pu le faire dans la leçon précédent .

15.Consulter l’historique des modifications Git :

La manière la plus simple de consulter l`historique des modifications Git est d`utiliser la commande git log.

16.Annuler des modifications apportées à un fichier :

Après un commit, on va continuer à travailler sur nos fichiers et à les modifier. On va pouvoir faire cela avec la commande générale git checkout – nom-du-fichier ou la nouvelle commande spécialisée git restore.

17.Qu’est ce qu’une branche ?

Git a une approche totalement différente des branches tant que Une branche, dans Git, est simplement un pointeur vers un commit . Le pointeur est un objet qui contient l`adresse d`une donnée stockée quelque part. Un commit est un objet qui contient un pointeur vers l`instantané du contenu indexé ainsi que des pointeurs vers le ou les commits le précédant directement.
